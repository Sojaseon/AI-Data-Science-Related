Cleaning (or pre-processing) the data typically consists of three steps.</p><div class="snippet-box snippet-box-ordered"><div><h2 class="title">How to Clean Your Data for NLP</h2><div class="description"><ol><li>Remove punctuation</li><li>Tokenize</li><li>Remove stop words</li><li>Stem</li><li>Lemmatize</li></ol></div></div></div><h3 dir="ltr">1. Remove Punctuation</h3><p dir="ltr">Punctuation can provide grammatical context to a sentence which supports human understanding. But for our vectorizer, which counts the number of words and not the context, punctuation does not add value. So we need to remove all special characters. For example, “How are you?” becomes: How are you<br /><br />Here’s how to do it:</p><img alt="nlp machine learning" data-entity-type="file" data-entity-uuid="d17eaa89-ec93-4247-9126-59d99042e62a" src="data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7" class="align-center b-lazy" width="1000" height="447" loading="lazy" data-src="https://builtin.com/sites/www.builtin.com/files/styles/ckeditor_optimize/public/inline-images/4_nlp%20machine%20learning_0.png" /><p dir="ltr">In <code>body_text_clean</code>, you can see we’ve removed all punctuation. I’ve becomes Ive and WILL!! Becomes WILL.</p><h3 dir="ltr">2.Tokenize</h3><p dir="ltr">Tokenizing separates text into units such as sentences or words. In other words, this function gives structure to previously unstructured text. For example: Plata o Plomo becomes ‘Plata’,’o’,’Plomo’.</p><figure><img alt="nlp machine learning" data-entity-type="file" data-entity-uuid="27991d49-23c3-4d13-906a-0a8464c0de1c" src="data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7" width="1000" height="443" loading="lazy" data-src="https://builtin.com/sites/www.builtin.com/files/styles/ckeditor_optimize/public/inline-images/5_nlp%20machine%20learning.png" class=" b-lazy" /><figcaption>Access raw code <a href="https://gist.github.com/BadreeshShetty/0861309de75358fd788235ff99a73aa5" target="_blank">here</a>.</figcaption></figure><p dir="ltr">In <code>body_text_tokenized</code>, we’ve generated all the words as tokens.</p><h3 dir="ltr">3. Remove Stop Words</h3><p dir="ltr">Stop words are common words that will likely appear in any text. They don’t tell us much about our data so we remove them. Again, these are words that are great for human understanding, but will confuse your machine learning program. For example: silver or lead is fine for me becomes silver, lead, fine.</p><figure><img alt="nlp machine learning" data-entity-type="file" data-entity-uuid="6d4d6aff-fa32-4226-ac41-5791fa779a9f" src="data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7" width="1000" height="481" loading="lazy" data-src="https://builtin.com/sites/www.builtin.com/files/styles/ckeditor_optimize/public/inline-images/6_nlp%20machine%20learning.png" class=" b-lazy" /><figcaption>Access raw code <a href="https://gist.github.com/BadreeshShetty/cc86f564e5b368c1ec9a6d19ecd682ff" target="_blank">here</a>.</figcaption></figure><p dir="ltr">In <code>body_text_nostop</code>, we remove all unnecessary words like “been,” “for,” and “the.”</p><h3 dir="ltr">4. Stem</h3><p dir="ltr">Stemming helps reduce a word to its stem form. It often makes sense to treat related words in the same way. It removes suffixes like “ing,” “ly,” “s” by a simple rule-based approach. Stemming reduces the corpus of words but often the actual words are lost, in a sense. For example: “Entitling” or  “Entitled” become  “Entitl.”</p><p dir="ltr"><em>Note: Some search engines treat words with the same stem as synonyms. </em></p><figure><img alt="nlp machine learning" data-entity-type="file" data-entity-uuid="92ee7226-c98e-4e48-94c7-64712ac6a0ae" src="data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7" width="1000" height="447" loading="lazy" data-src="https://builtin.com/sites/www.builtin.com/files/styles/ckeditor_optimize/public/inline-images/7_nlp%20machine%20learning.png" class=" b-lazy" /><figcaption>Access raw code <a href="https://gist.github.com/BadreeshShetty/3bd45483ebc032dc768caaef9ea77c1b" target="_blank">here</a>.</figcaption></figure><p dir="ltr">In <code>body_text_stemmed</code>, words like entry and goes are stemmed to entri and goe even though they don’t mean anything in English.</p><h3 dir="ltr">5. Lemmatize</h3><p dir="ltr">Lemmatizing derives the root form (“lemma”) of a word. This practice is more robust than stemming because it uses a dictionary-based approach (i.e a morphological analysis) to the root word. For example, “Entitling” or “Entitled” become “Entitle.”</p><p dir="ltr">In short, stemming is typically faster as it simply chops off the end of the word, but without understanding the word’s context. Lemmatizing is slower but more accurate because it takes an informed analysis with the word’s context in mind.</p><figure><img alt="nlp machine learning" data-entity-type="file" data-entity-uuid="f8e6fe34-2b3a-4bcc-975b-9403f2fb787c" src="data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7" width="1090" height="506" loading="lazy" data-src="https://builtin.com/sites/www.builtin.com/files/styles/ckeditor_optimize/public/inline-images/8_nlp%20machine%20learning.png" class=" b-lazy" /><figcaption>Access raw code <a href="https://gist.github.com/BadreeshShetty/d421a71ab1ce7c25f04ae651171d0aa0" target="_blank">here</a>.</figcaption></figure><p dir="ltr">In <code>body_text_stemmed</code>, we can see words like “chances” are lemmatized to “chance” but stemmed to “chanc.”</p><p class="bix-embed-read-more"><span><span>Want More Data Science Tutorials?</span><a href="https://builtin.com/data-science/automate-data-analysis" target="_blank">Need to Automate Your Data Analysis? Bag-Of-Words</h3><p dir="ltr">Bag-of-Words (BoW) or CountVectorizer describes the presence of words within the text data. This process gives a result of one if present in the sentence and zero if absent. This model therefore, creates a bag of words with a document-matrix count in each text document.</p><pre><code class="language-python">from sklearn.feature_extraction.text import CountVectorizer

count_vect = CountVectorizer(analyzer=clean_text)
X_counts = count_vect.fit_transform(data['body_text'])

print(count_vect.get_feature_names())</code></pre><p dir="ltr">We apply BoW to the <code>body_text</code> so the count of each word is stored in the document matrix. N-Grams</h3><p dir="ltr">N-grams are simply all combinations of adjacent words or letters of length <code>n</code> that we find in our source text. N-grams with <code>n=1</code> are called unigrams, <code>n=2</code> are bigrams, and so on. </p><figure><img alt="nlp machine learning" data-entity-type="file" data-entity-uuid="c3e92ac6-74c9-42e5-b170-4c5355a8e4e1" src="data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7" width="804" height="168" loading="lazy" data-src="https://builtin.com/sites/www.builtin.com/files/styles/ckeditor_optimize/public/inline-images/9_nlp%20machine%20learning.png" class=" b-lazy" /><figcaption>Access raw code <a href="https://gist.github.com/BadreeshShetty/0dc2305ac012a5018aa39879489617be" target="_blank">here</a>.</figcaption></figure><p dir="ltr">Unigrams usually don’t contain much information as compared to bigrams or trigrams. The basic principle behind N-grams is that they capture which letter or word is likely to follow a given word. The longer the N-gram (higher <code>n</code>), the more context you have to work with.</p><pre><code class="language-python">from sklearn.feature_extraction.text import CountVectorizer

ngram_vect = CountVectorizer(ngram_range=(2,2),analyzer=clean_text) # It applies only bigram vectorizer

print(X_counts.shape)
print(ngram_vect.get_feature_names())</code></pre><p>We’ve applied N-Gram to the <code>body_text</code>, so the count of each group of words in a sentence is stored in the document matrix. (Check the <a href="https://github.com/BadreeshShetty/Natural-Language-Processing-NLP-for-Machine-Learning" target="_blank">repo</a>).</p><h3 dir="ltr">3. TF-IDF</h3><p dir="ltr">TF-IDF computes the relative frequency with which a word appears in a document compared to its frequency across all documents. It’s more useful than term frequency for identifying key words in each document (high frequency in that document, low frequency in other documents).</p><p dir="ltr"><em>Note: We use TF-IDF for search engine scoring, text summarization and document clustering. Check my article on <a href="https://builtin.com/data-science/recommender-systems" target="_blank">recommender systems</a>  to learn more about TF-IDF.</em></p><pre><code class="language-python">from sklearn.feature_extraction.text import TfidfVectorizer

tfidf_vect = TfidfVectorizer(analyzer=clean_text)

print(X_tfidf.shape)
print(tfidf_vect.get_feature_names())</code></pre><p dir="ltr">We’ve applied TF-IDF in the body_text, so the relative count of each word in the sentences is stored in the document matrix.
